# 开发板环境修复指南

## 🚨 问题诊断

根据错误信息，您遇到了两个主要问题：

1. **APT软件源问题**：Debian Buster软件源404错误
2. **PyAudio编译失败**：缺少portaudio.h头文件

## 🔧 解决方案

### 方案1：修复软件源（推荐）

#### 1.1 更新软件源列表

```bash
# 备份原始源列表
sudo cp /etc/apt/sources.list /etc/apt/sources.list.backup

# 编辑软件源
sudo nano /etc/apt/sources.list
```

#### 1.2 替换为可用的Debian源

将内容替换为以下之一：

**选项A：使用Debian官方档案源**
```bash
# Debian Buster 官方档案源
deb http://archive.debian.org/debian buster main
deb http://archive.debian.org/debian-security buster/updates main
deb http://archive.debian.org/debian buster-updates main

# Rock-chips源保持不变
deb http://repo.rock-chips.com/debian buster main
```

**选项B：使用中科大镜像源**
```bash
# 中科大镜像源
deb http://mirrors.ustc.edu.cn/debian buster main contrib non-free
deb http://mirrors.ustc.edu.cn/debian buster-updates main contrib non-free
deb http://mirrors.ustc.edu.cn/debian-security buster/updates main contrib non-free

# Rock-chips源保持不变
deb http://repo.rock-chips.com/debian buster main
```

**选项C：使用清华大学镜像源**
```bash
# 清华大学镜像源
deb https://mirrors.tuna.tsinghua.edu.cn/debian buster main contrib non-free
deb https://mirrors.tuna.tsinghua.edu.cn/debian buster-updates main contrib non-free
deb https://mirrors.tuna.tsinghua.edu.cn/debian-security buster/updates main contrib non-free

# Rock-chips源保持不变
deb http://repo.rock-chips.com/debian buster main
```

#### 1.3 更新软件包列表

```bash
# 更新包列表
sudo apt update

# 如果仍有问题，可以忽略GPG检查（临时解决）
sudo apt update --allow-releaseinfo-change
```

### 方案2：安装PyAudio依赖

#### 2.1 安装系统依赖

```bash
# 安装portaudio开发库
sudo apt install -y portaudio19-dev python3-dev

# 如果上面失败，尝试这些替代方案
sudo apt install -y libportaudio2 libportaudiocpp0 libportaudio-dev

# 安装其他可能需要的依赖
sudo apt install -y build-essential python3-pip
```

#### 2.2 安装PyAudio

```bash
# 方法1：使用apt安装预编译版本
sudo apt install python3-pyaudio

# 方法2：如果apt版本不可用，使用pip
pip3 install pyaudio

# 方法3：指定portaudio路径编译
pip3 install --global-option="build_ext" --global-option="-I/usr/include" --global-option="-L/usr/lib" pyaudio
```

### 方案3：无音频模式运行（备用方案）

如果音频安装仍然失败，可以修改集成系统以在无音频模式下运行：

#### 3.1 创建无音频版本

```python
# 在board_integrated_system.py开头强制禁用音频
HAS_AUDIO = False

# 或者设置环境变量
import os
os.environ['DISABLE_AUDIO'] = '1'
```

#### 3.2 安装其他依赖

```bash
# 安装基础依赖
pip3 install opencv-python numpy

# 安装pygame（如果需要图形界面）
pip3 install pygame

# 安装网络相关
pip3 install requests
```

## 🛠️ 完整修复脚本

创建一个自动修复脚本：

```bash
#!/bin/bash
# 保存为 fix_environment.sh

echo "🔧 开始修复开发板环境..."

# 1. 备份原始源
sudo cp /etc/apt/sources.list /etc/apt/sources.list.backup.$(date +%Y%m%d)

# 2. 写入新的软件源
sudo tee /etc/apt/sources.list > /dev/null <<EOF
# Debian Buster 档案源
deb http://archive.debian.org/debian buster main contrib non-free
deb http://archive.debian.org/debian-security buster/updates main contrib non-free

# Rock-chips源
deb http://repo.rock-chips.com/debian buster main
EOF

# 3. 更新包列表
echo "📦 更新软件包列表..."
sudo apt update --allow-releaseinfo-change

# 4. 安装基础开发工具
echo "🔨 安装基础开发工具..."
sudo apt install -y build-essential python3-dev python3-pip

# 5. 安装音频相关依赖
echo "🎵 安装音频依赖..."
sudo apt install -y portaudio19-dev libasound2-dev || echo "⚠️ 音频依赖安装失败，将使用无音频模式"

# 6. 安装Python包
echo "🐍 安装Python依赖..."
pip3 install opencv-python numpy requests

# 尝试安装PyAudio
echo "🎤 尝试安装PyAudio..."
pip3 install pyaudio || sudo apt install -y python3-pyaudio || echo "⚠️ PyAudio安装失败，语音功能将不可用"

# 尝试安装pygame
echo "🎮 尝试安装pygame..."
pip3 install pygame || echo "⚠️ pygame安装失败，将使用控制台模式"

echo "✅ 环境修复完成！"
echo "📝 请运行 python3 board_integrated_system.py 测试系统"
```

#### 使用修复脚本

```bash
# 创建并运行修复脚本
nano fix_environment.sh
chmod +x fix_environment.sh
./fix_environment.sh
```

## 🔄 替代方案

### 方案A：使用Docker容器

```bash
# 创建Dockerfile
cat > Dockerfile <<EOF
FROM debian:buster
RUN apt update && apt install -y \\
    python3 python3-pip \\
    python3-opencv \\
    python3-pyaudio \\
    python3-pygame
WORKDIR /app
COPY . .
CMD ["python3", "board_integrated_system.py"]
EOF

# 构建和运行
docker build -t medical-system .
docker run --device=/dev/video0 medical-system
```

### 方案B：使用conda环境

```bash
# 安装miniconda
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-aarch64.sh
bash Miniconda3-latest-Linux-aarch64.sh

# 创建环境
conda create -n medical python=3.8
conda activate medical
conda install opencv pyaudio pygame -c conda-forge
```

## 📝 注意事项

1. **软件源选择**：选择网络延迟最低的镜像源
2. **依赖优先级**：优先使用apt安装，再考虑pip
3. **权限问题**：某些操作需要sudo权限
4. **网络问题**：确保开发板网络连接正常
5. **存储空间**：确保有足够的磁盘空间

## 🆘 如果问题仍然存在

### 检查系统信息

```bash
# 检查系统版本
cat /etc/os-release
uname -a

# 检查Python版本
python3 --version

# 检查已安装的包
pip3 list | grep -E "(opencv|pyaudio|pygame)"

# 检查摄像头设备
ls -la /dev/video*
```
