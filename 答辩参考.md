# 医疗诊断系统答辩参考

## 一、系统整体介绍

### 1.1 系统概述
我们的系统是一个基于YOLO目标检测模型的眼部疾病智能诊断系统，采用开发板-PC端协同架构。开发板端负责图像采集、语音交互和触摸控制，PC端负责AI模型推理和结果分析。两部分通过WiFi网络进行高速数据传输，形成一个完整的智能医疗诊断解决方案。

系统主要功能包括：
1. 实时摄像头图像采集与传输
2. 基于YOLOv11的AI眼部疾病识别
3. 语音交互功能（语音识别与合成）
4. 触摸屏控制界面
5. 网络通信与数据传输
6. 诊断结果展示与建议生成
7. 屏幕共享与远程控制

### 1.2 技术架构
系统采用分布式架构，开发板端使用OpenCV进行图像采集，通过UDP协议将图像传输到PC端。PC端使用YOLOv11模型进行眼部疾病识别，并将结果返回给开发板。

系统架构图：
```
开发板端                     PC端
┌─────────────────┐         ┌─────────────────┐
│ 摄像头捕获      │         │ 图像接收        │
│ ↓               │         │ ↓               │
│ 图像增强        │ ──WiFi─→ │ AI诊断分析      │
│ ↓               │         │ ↓               │
│ 网络传输        │         │ 结果生成        │
│ ↓               │         │ ↓               │
│ 结果显示        │ ←─WiFi── │ 结果发送        │
└─────────────────┘         └─────────────────┘

触摸屏控制 ←─────── 延迟优化 ─────────→ 屏幕投影
```

主要技术栈：
- 开发板端：Python, OpenCV, PyAudio, Pygame
- PC端：Python, PyQt5, YOLOv11 (Ultralytics 8.3.93), DeepSeek API
- 通信协议：UDP
- AI模型：YOLOv11

### 1.3 系统特点
1. **分布式处理**：将计算密集型任务部署在PC端，降低开发板硬件要求
2. **多模态交互**：支持图像识别、语音交互和触摸控制三种交互方式
3. **实时性优化**：采用UDP协议和数据分片技术，保证传输实时性
4. **模块化设计**：各功能模块独立设计，便于维护和扩展
5. **容错机制**：具备网络断线重连和错误恢复功能

## 二、可能被问到的问题及参考答案

### 2.1 系统功能相关问题

#### Q: 你们的系统解决了什么实际问题？
A: 我们的系统主要解决了基层医疗机构缺乏专业眼科医生的问题。通过AI技术，系统可以自动识别多种常见眼部疾病，如白内障、青光眼、糖尿病视网膜病变等，为偏远地区或医疗资源匮乏地区提供初步诊断支持。用户只需拍摄眼部照片，系统就能给出可能的疾病类型和治疗建议，大大降低了眼科疾病诊断的门槛。

#### Q: 系统的准确率如何？
A: 我们的系统基于YOLOv11模型进行训练，在测试集上的准确率达到了90%以上（根据实际训练结果填写）。为了进一步提高诊断准确性，我们还集成了DeepSeek大模型API，可以根据检测结果生成专业的医疗建议。需要注意的是，我们的系统定位是辅助诊断工具，不能替代专业医生的诊断。

#### Q: 为什么选择YOLOv11模型？
A: 我们选择YOLOv11主要有以下几个原因：
1. YOLOv11在目标检测任务中具有较高的准确率和较快的推理速度
2. 模型相对轻量化，适合在PC端运行
3. Ultralytics提供了完善的工具链，便于模型训练和部署
4. 社区支持活跃，文档齐全，便于开发和调试
5. 我们使用的版本是Ultralytics 8.3.93，这是一个稳定且功能完善的版本

### 2.2 技术实现相关问题

#### Q: 开发板和PC端是如何通信的？
A: 开发板和PC端通过UDP协议进行通信，使用多个端口分别处理不同类型的数据：
1. 摄像头端口(5002)：传输图像数据
2. 诊断端口(5003)：传输诊断结果
3. 命令端口(5004)：传输控制命令
4. 语音端口(5005/5006)：传输语音数据
5. 触摸控制端口(5001)：传输触摸控制指令

我们选择UDP协议主要是考虑到实时性要求，特别是在视频流传输方面，UDP的低延迟特性更适合我们的应用场景。

#### Q: 图像传输过程中如何保证数据完整性？
A: 我们采用了分片传输和重组的机制来保证图像数据的完整性：
1. 大图像数据会被分片传输，每个分片包含包头信息（包索引、总包数等）
2. 接收端根据包头信息对分片进行排序和重组
3. 通过校验机制确保所有分片都已接收到
4. 如果某个分片丢失，系统会请求重传

#### Q: 语音交互是如何实现的？
A: 语音交互分为两个部分：
1. 语音识别：开发板端通过麦克风采集音频，使用本地的语音识别引擎（如Vosk）或通过网络发送到PC端进行识别
2. 语音合成：PC端将文本信息通过TTS技术转换为语音，再传输回开发板进行播放

整个过程实现了双向语音交互，用户可以通过语音与系统进行对话。

### 2.3 创新点相关问题

#### Q: 你们系统的主要创新点是什么？
A: 我们系统的创新点主要体现在以下几个方面：
1. 硬件集成创新：将AI算法部署在PC端，通过网络与开发板连接，既保证了计算能力又降低了开发板的硬件要求

4. 界面友好：开发了直观的图形界面，操作简单，便于非专业人员使用

#### Q: 与市面上类似产品相比，你们的优势在哪里？
A: 与市面上类似产品相比，我们的优势包括：

#### Q: 你们的系统适用于哪些场景？
A: 我们的系统主要适用于以下场景：
1. 基层医疗机构：为缺乏专业眼科医生的地区提供初步诊断支持

A: 在数据安全方面，我们采取了以下措施：
1. 数据传输加密：所有网络传输的数据都经过加密处理
2. 本地存储：敏感数据优先存储在本地，减少网络传输
3. 权限控制：系统设有访问权限控制，防止未授权访问
4. 隐私保护：系统不会收集用户的个人身份信息，所有数据仅用于诊断目的

## 六、系统技术细节和优化策略

### 6.1 图像处理优化
为了提高诊断准确性，我们在图像处理方面做了多项优化：
1. 图像增强：使用双边滤波去噪、锐化处理和CLAHE对比度增强
2. 分辨率适配：将图像统一调整为640x480像素，平衡识别精度和处理速度
3. JPEG压缩：在传输前对图像进行压缩，减少网络传输压力

### 6.2 网络传输优化
针对网络传输，我们采用了以下优化策略：
1. UDP协议：相比TCP，UDP具有更低的延迟，更适合实时视频传输
2. 数据分片：大图像数据分片传输，避免单个数据包过大导致的传输失败
3. 流量控制：控制视频流帧率，避免网络拥塞

### 6.3 AI模型优化
在AI模型方面，我们进行了以下优化：
1. 模型选择：选择YOLOv11s模型，在精度和速度之间取得平衡
2. 模型量化：对模型进行量化处理，减少模型大小和推理时间
3. 后处理优化：优化NMS等后处理算法，提高检测结果的准确性

### 6.4 系统性能优化
为了提升系统整体性能，我们实施了以下优化措施：
1. 多线程处理：使用多线程技术处理不同的任务，避免阻塞
2. 资源管理：合理管理内存和CPU资源，避免资源泄漏
3. 延迟加载：对非核心组件进行延迟加载，提高系统启动速度

- 使用UDP协议减少传输延迟
- 图像压缩减少传输时间
