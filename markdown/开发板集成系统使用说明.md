# 开发板集成医疗诊断系统使用说明

## 📋 系统概述

这是一个专为开发板设计的集成医疗诊断系统，可以在JupyterLab环境中单文件运行，具备完整的摄像头、语音交互和PC端通信功能。

### 🎯 核心功能

- **摄像头图像捕获**: 实时预览、拍照、图像增强
- **AI医疗诊断**: 与PC端系统交互，获取诊断结果
- **语音交互**: 录音、语音识别、语音合成播放
- **触摸屏控制**: 图形界面或控制台模式
- **网络通信**: 多端口数据传输
- **JupyterLab兼容**: 单文件运行，依赖自检

## 🔧 环境要求

### 必需依赖
```bash
pip3 install opencv-python numpy
```

### 可选依赖
```bash
# 音频功能（推荐）
pip3 install pyaudio

# 图形界面（可选）
pip3 install pygame
```

### 系统要求
- Python 3.6+
- OpenCV支持的USB摄像头
- 网络连接（与PC端通信）

## 🎮 操作指南

### 图形界面模式（如果pygame可用）

#### 触摸按钮
- **拍照诊断**: 拍摄照片并发送到PC端进行AI诊断
- **语音对话**: 开始录音并与PC端进行语音交互
- **摄像头预览**: 启动实时摄像头预览窗口

#### 状态显示
- 摄像头状态（运行中/已停止）
- 音频功能状态（可用/不可用）
- PC端连接信息

### 控制台模式（fallback模式）

#### 可用命令
```
c - 启动摄像头预览
p - 拍照并诊断
r - 开始录音
s - 显示系统状态
q - 退出系统
```

### 摄像头预览模式

#### 快捷键操作
```
空格键 - 拍照并发送诊断
s      - 仅保存照片
r      - 开始录音对话
q      - 退出预览
```

## 🔄 系统工作流程

### 1. 图像诊断流程
```
拍照 → 图像增强 → 保存本地 → 发送PC端 → 等待诊断 → 显示结果
```

### 2. 语音交互流程
```
录音 → 编码传输 → PC端识别 → AI处理 → 语音合成 → 播放回复
```

### 3. 网络通信流程
```
开发板 ←→ PC端
多端口并发通信
- 图像数据: 5002
- 诊断结果: 5003
- 控制命令: 5004
- 语音数据: 5005/5006
- 触摸控制: 5001
```

## 🛠️ 故障排除

### 摄像头问题
```python
# 检查摄像头状态
camera_status = camera_manager.get_camera_status()
print(camera_status)

# 常见解决方法
- 检查USB连接
- 确认摄像头权限
- 重启摄像头服务
```

### 音频问题
```bash
# 安装音频依赖
sudo apt install python3-pyaudio

# 或者使用pip
pip3 install pyaudio

# 如果编译失败，安装开发包
sudo apt install portaudio19-dev python3-dev
```

### 网络连接问题
```python
# 检查PC端IP是否正确
PC_IP = "你的PC端实际IP"

# 检查端口是否被占用
netstat -tuln | grep 500[0-9]

# 防火墙设置
sudo ufw allow 5000:5010/udp
```

### 依赖缺失处理
系统具备优雅降级功能：
- **无PyAudio**: 禁用音频功能，其他功能正常
- **无Pygame**: 使用控制台模式
- **无OpenCV**: 系统无法启动（必需依赖）

### 网络安全
- 确保PC端和开发板在同一安全网络

- 定期清理敏感数据
- 确保数据传输加密（如需要）
