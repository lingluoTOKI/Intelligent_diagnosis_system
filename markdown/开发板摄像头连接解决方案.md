## 🚀 使用指南

### 第一步：配置PC端IP地址

1. **查看PC端实际IP地址**：
   ```bash
   # Windows
   ipconfig
   
   # Linux/Mac
   ifconfig
   ```

2. **修改开发板代码中的PC_IP**：
   ```python
   # 在 board_integrated_system.py 第78行附近
   PC_IP = "你的PC实际IP地址"  # 例如: "192.168.1.100"
   ```

### 第二步：启动PC端程序

确保PC端的医疗诊断系统已启动并监听以下端口：
- 📹 摄像头数据端口: `5002`
- 🔍 诊断结果端口: `5003`
- ⚙️ 命令控制端口: `5004`

### 第三步：运行连接测试

在开发板上运行测试脚本：
```bash
python test_board_connection.py
```

这个测试脚本会检查：
- 网络连通性 (ping测试)
- 摄像头功能
- PC端通信
- 图像传输

### 第四步：启动开发板系统

```bash
python board_integrated_system.py
```

系统启动后会：
1. 自动检测PC端连接
2. 如果连接成功，自动开始视频流传输
3. 提供图形界面或控制台界面进行操作

## ⚠️ 故障排除

### 问题1：网络连接失败

**现象**: 显示"❌ [网络] 连接测试超时"

**解决方案**:
1. 检查PC端程序是否已启动
2. 确认PC和开发板在同一局域网
3. 检查防火墙设置：
   ```bash
   # Windows防火墙添加端口规则
   netsh advfirewall firewall add rule name="Medical System" dir=in action=allow protocol=UDP localport=5002,5003,5004
   ```
4. 验证IP地址是否正确

### 问题2：摄像头无法打开

**现象**: 显示"❌ [摄像头] 无法打开摄像头设备"

**解决方案**:
1. 检查摄像头是否正确连接
2. 确认摄像头权限：
   ```bash
   # Linux系统
   sudo chmod 666 /dev/video0
   ```
3. 检查是否有其他程序占用摄像头

### 问题3：视频流传输卡顿

**现象**: PC端接收到的视频流不流畅

**解决方案**:
1. 调整视频流帧率：
   ```python
   self.stream_fps = 10  # 降低到10FPS
   ```
2. 调整图像压缩质量：
   ```python
   cv2.imencode('.jpg', frame, [cv2.IMWRITE_JPEG_QUALITY, 50])  # 降低质量
   ```
3. 检查网络带宽

### 问题4：端口被占用

**现象**: 显示端口绑定失败

**解决方案**:
```bash
# 检查端口占用
netstat -ano | findstr :5002
netstat -ano | findstr :5003
netstat -ano | findstr :5004

# 终止占用进程
taskkill /PID <进程ID> /F
```
