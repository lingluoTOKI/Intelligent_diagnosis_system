# 开发板-PC端整合医疗诊断系统使用说明

## 🎯 系统功能概述

本系统整合了开发板和PC端的医疗诊断功能，实现了以下三个核心交互：

### 1. 📸 拍照诊断流程
- **开发板**：调用摄像头拍照
- **传输**：图像数据传输到PC端
- **PC端**：AI模型识别分析疾病
- **结果**：诊断结果返回开发板显示

### 2. 🔊 AI建议播报
- **PC端**：生成医疗建议
- **选择**：可选择播放设备（PC端/开发板/双端）
- **播放**：TTS语音播报建议内容

### 3. 🎤 语音对话
- **开发板**：麦克风阵列录音
- **传输**：语音数据/文本传给PC端
- **PC端**：DeepSeek大模型处理
- **回复**：AI回复可选择播放设备

## 🚀 快速开始

### 第一步：启动PC端系统
```bash
python visualization_test2.py
```

### 第二步：启动开发板系统
```bash
python board_integrated_system.py
```

### 第三步：配置音频设备
1. 在PC端点击"开发板交互管理"
2. 选择音频播放设备：
   - **PC端播放**：仅在PC端播放
   - **开发板播放**：仅在开发板播放
   - **双端播放**：同时在两端播放
3. 点击"测试音频播放"验证设置

## 📋 详细使用步骤

### 拍照诊断操作
1. **开发板端**：
   - 触摸"拍照"按钮或按空格键
   - 系统自动拍照并传输到PC端
   
2. **PC端**：
   - 自动接收图像并进行AI诊断
   - 显示诊断结果和医疗建议
   - 根据音频设备设置播放建议

3. **结果查看**：
   - 开发板屏幕显示诊断结果
   - PC端保存诊断历史记录

### 语音对话操作
1. **开发板端**：
   - 触摸"录音"按钮或按'r'键
   - 录音5秒钟（可配置）
   - 系统进行语音识别并发送文本

2. **PC端**：
   - 接收语音文本
   - DeepSeek AI处理并生成回复
   - 根据设置选择播放设备

3. **回复播放**：
   - PC端/开发板根据设置播放TTS
   - PC端聊天界面显示对话记录

## ⚙️ 配置说明

### 网络配置
- 确保开发板和PC在同一网络
- 默认端口配置：
  - 摄像头数据：5002
  - 诊断结果：5003
  - 命令控制：5004
  - 语音数据：5005-5006

### 音频设备设置
- **PC端播放**：使用PC端扬声器/耳机
- **开发板播放**：使用开发板扬声器
- **双端播放**：同时在两端播放（适合演示）

### AI模型配置
- 确保PC端已加载AI诊断模型
- 配置DeepSeek API密钥
- 测试网络连接状态

## 🔧 故障排除

### 连接问题
- 检查网络连接状态
- 确认IP地址配置正确
- 测试端口是否被占用

### 音频问题
- 检查音频设备是否可用
- 确认TTS引擎正常工作
- 测试麦克风权限设置

### AI诊断问题
- 确认AI模型已正确加载
- 检查DeepSeek API配置
- 验证图像传输完整性

## 🎯 使用技巧

1. **最佳拍照效果**：
   - 保持摄像头稳定
   - 确保光线充足
   - 拍摄对象居中清晰

2. **语音对话技巧**：
   - 说话清晰，语速适中
   - 避免环境噪音干扰
   - 使用医疗相关词汇

3. **音频设备选择**：
   - 演示时选择"双端播放"
   - 私人使用选择"PC端播放"
   - 开发板独立使用选择"开发板播放"
