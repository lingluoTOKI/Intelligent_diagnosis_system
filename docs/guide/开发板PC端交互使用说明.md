# 开发板与PC端交互使用说明

## 🎯 系统概述

这是一个完整的智能医疗诊断系统，包含开发板端和PC端两个部分，通过UDP网络协议进行实时数据交互。

### 系统架构
```
开发板端 (board_integrated_system.py) ←→ PC端 (visualization_test2.py)
├── 图像诊断: 开发板拍照 → PC端AI分析 → 返回诊断结果
├── 语音交互: 开发板录音 → PC端语音识别 → 返回语音回复
├── 实时监控: 开发板摄像头 → PC端实时预览
└── 状态同步: 心跳检测、连接状态监控
```

## 🚀 快速启动

### 1. 启动PC端系统
```bash
# 在PC端运行
python visualization_test2.py
```

**启动后操作：**
1. 在PC端界面点击"开发板交互"按钮
2. 系统会自动启动命令监听器和摄像头接收器
3. 等待开发板连接

### 2. 启动开发板系统
```bash
# 在开发板上运行
python board_integrated_system.py
```

**启动后操作：**
1. 系统会自动初始化摄像头、音频和网络模块
2. 启动触摸界面（如果有图形支持）或控制台界面
3. 自动发送心跳包到PC端

## 🔧 交互功能详解

### 图像诊断流程

#### 开发板端操作
1. **拍照诊断**: 在触摸界面点击"拍照诊断"按钮，或按空格键
2. **图像处理**: 系统自动进行图像增强（去噪、锐化、色彩增强）
3. **数据发送**: 图像被压缩并分片发送到PC端

#### PC端处理
1. **接收图像**: 摄像头接收器接收分片数据并重组
2. **AI诊断**: 使用YOLO模型进行眼部疾病检测
3. **结果返回**: 将诊断结果和建议发送回开发板

#### 数据流向
```
开发板摄像头 → 图像增强 → 分片压缩 → UDP发送(5002端口)
                                    ↓
PC端接收器 ← 数据重组 ← 图像解码 ← UDP接收(5002端口)
                                    ↓
AI模型分析 → 疾病检测 → 结果生成 → UDP发送(5003端口)
                                    ↓
开发板接收 ← 结果解析 ← JSON解码 ← UDP接收(5003端口)
```

### 语音交互流程

#### 开发板端操作
1. **语音录音**: 点击"语音对话"按钮或按R键
2. **音频处理**: 录制5秒音频，转换为WAV格式
3. **数据发送**: 音频数据分片发送到PC端

#### PC端处理
1. **接收音频**: 语音接收器接收音频数据
2. **语音识别**: 使用Vosk模型进行语音识别
3. **智能回复**: 生成相应的语音回复

#### 数据流向
```
开发板麦克风 → 音频录制 → WAV编码 → 分片发送(5005端口)
                                    ↓
PC端接收器 ← 数据重组 ← 音频解码 ← UDP接收(5005端口)
                                    ↓
语音识别 → 文本生成 → TTS合成 → UDP发送(5006端口)
                                    ↓
开发板扬声器 ← 音频播放 ← 数据接收 ← UDP接收(5006端口)
```

### 实时监控流程

#### 开发板端
1. **摄像头预览**: 点击"摄像头预览"按钮或按C键
2. **实时显示**: 显示摄像头画面和操作提示
3. **触摸控制**: 支持触摸屏操作和键盘快捷键

#### PC端
1. **实时接收**: 持续接收开发板摄像头数据
2. **画面显示**: 在PC端界面显示实时预览
3. **状态监控**: 监控连接状态和数据传输质量

## 📡 网络协议详解

### 端口配置
| 端口 | 用途 | 数据流向 | 协议 |
|------|------|----------|------|
| 5002 | 图像数据传输 | 开发板 → PC端 | UDP |
| 5003 | 诊断结果返回 | PC端 → 开发板 | UDP |
| 5004 | 命令控制 | 双向通信 | UDP |
| 5005 | 语音数据发送 | 开发板 → PC端 | UDP |
| 5006 | 语音合成返回 | PC端 → 开发板 | UDP |
| 5007 | 语音命令控制 | 开发板 → PC端 | UDP |

### 数据包格式

#### 图像数据包
```
[4字节请求ID哈希][2字节包索引][2字节总包数][1字节标志位][图像数据]
```

#### 语音数据包
```
[8字节语音ID][2字节包索引][2字节总包数][1字节标志位][音频数据]
```

#### 命令数据包
```json
{
    "type": "命令类型",
    "timestamp": "时间戳",
    "data": "命令数据",
    "params": "参数"
}
```

### 心跳机制
- **频率**: 每5秒发送一次心跳包
- **内容**: 包含设备状态和运行信息
- **响应**: PC端自动回复心跳响应
- **监控**: 用于检测连接状态和网络延迟

## 🔍 故障排除

### 常见问题

#### 1. 网络连接失败
**现象**: 开发板显示"网络连接失败"
**排查步骤**:
```bash
# 检查IP配置
ping 172.20.10.3

# 检查端口占用
netstat -tuln | grep 500[0-9]

# 测试UDP连接
nc -u 172.20.10.3 5002
```
**解决方案**:
- 修改`PC_IP`为正确的PC端IP地址
- 确保两设备在同一网络
- 检查防火墙设置

#### 2. 图像传输失败
**现象**: 图像发送失败或诊断超时
**排查步骤**:
```python
# 检查图像编码
import cv2
test_img = np.zeros((480, 640, 3), dtype=np.uint8)
_, encoded = cv2.imencode('.jpg', test_img)
print(f"编码后大小: {len(encoded.tobytes())} 字节")
```
**解决方案**:
- 检查摄像头是否正常工作
- 验证图像编码格式
- 确认PC端服务器正在监听

#### 3. 诊断结果接收失败
**现象**: 无法接收诊断结果
**排查步骤**:
```bash
# 检查开发板监听端口
netstat -tuln | grep 5003

# 测试PC端发送
echo '{"type":"test"}' | nc -u 开发板IP 5003
```
**解决方案**:
- 确认开发板正确绑定5003端口
- 检查PC端是否正确发送结果
- 验证JSON格式是否正确
